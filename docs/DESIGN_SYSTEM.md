# ECHO — Design System

> This document defines ECHO's visual language, component patterns, and design rules. It is the authoritative reference for all UI work — whether generated by AI tools or written by human developers. The goal: every screen in ECHO should feel like it belongs to the same product.

**Design DNA:** Bold, modern, confident. Inspired by [Attio](https://attio.com) — dense but breathable data views, subtle gradients, tactile hover states, smooth animations, and a product that feels like a creative tool rather than a spreadsheet. ECHO should feel premium, fast, and alive.

## 1. Design Principles

These five principles guide every design decision in ECHO. When in doubt, refer back to these.

### 1.1 Dense but Breathable

ECHO is a data-heavy application. Agents work with tables of 50+ rows, detail pages with 30+ fields, and pipelines with multiple stages. The UI must pack information tightly without feeling cramped. Achieve this through consistent spacing, clear visual hierarchy, and generous use of whitespace between sections (not between every element).

### 1.2 Bold, Not Loud

Use strong visual contrast, confident typography, and saturated accent colors — but sparingly and with purpose. Every bold element should direct attention to what matters. If everything is bold, nothing is.

### 1.3 Interactive Surfaces

Every interactive element should feel tactile. Hover states, pressed states, and transitions should give the user confidence that the interface is responding to them. Buttons should feel like buttons. Rows should highlight on hover. Drag handles should invite grabbing. Like Attio: the UI should feel alive, not static.

### 1.4 Progressive Disclosure

Show the essential information first. Reveal complexity on demand through expandable sections, slide-out panels, hover tooltips, and drill-down detail pages. The listing table shows key columns; the detail page reveals everything.

### 1.5 Systematic Consistency

The same component should look and behave the same everywhere. A status badge on the listing table should look identical to a status badge on the deal table. A comment section on a listing should feel identical to a comment section on a deal. Build once, reuse everywhere.

### 2.6 Color Usage Rules

- **Primary orange** is reserved for: primary action buttons, active navigation states, selected tabs, toggle switches (on state), links, and the brand logo.
- **Never** use primary orange for large background fills in the dashboard. It's an accent, not a surface.
- **Status colors** are the only way to communicate state. Don't use orange for success or blue for errors.
- **Text** should almost always be `stone-800` (primary) or `stone-500` (secondary). Never use pure black (`#000`) for text.
- **Borders** are `stone-200` in light mode, `stone-700` in dark mode.
- **Hover backgrounds** use the `50` shade of the relevant color (e.g., `orange-50` for hovering a primary element, `stone-50` for hovering a neutral row).

## 3. Typography

### 3.1 Font Family

```
Font Stack:
  Sans (Latin):  'Inter', system-ui, -apple-system, sans-serif
  Sans (Thai):   'IBM Plex Sans Thai', 'Noto Sans Thai', sans-serif
  Mono:          'JetBrains Mono', 'Fira Code', monospace
```

**Inter** is the primary font for Latin characters. It's highly legible at small sizes (important for data tables), has excellent number alignment (tabular figures), and is the standard for modern SaaS. Load from Google Fonts or self-host.

**IBM Plex Sans Thai** is the dedicated font for Thai text. It pairs well with Inter — both are geometric, clean, and designed for UI readability. It supports all Thai glyphs and maintains consistent vertical rhythm when mixed with Latin characters. Load from Google Fonts.

**JetBrains Mono** is used only for: listing IDs, code snippets, and numerical data where alignment matters (prices, sizes).

**Font loading (Next.js):**
```tsx
import { Inter } from 'next/font/google'
import { IBM_Plex_Sans_Thai } from 'next/font/google'

const inter = Inter({
  subsets: ['latin'],
  variable: '--font-inter',
})

const ibmPlexSansThai = IBM_Plex_Sans_Thai({
  subsets: ['thai'],
  weight: ['400', '500', '600', '700'],
  variable: '--font-ibm-plex-thai',
})
```

**Tailwind config:**
```js
fontFamily: {
  sans: ['var(--font-inter)', 'var(--font-ibm-plex-thai)', 'system-ui', 'sans-serif'],
  mono: ['JetBrains Mono', 'Fira Code', 'monospace'],
}
```

The browser will automatically fall back to IBM Plex Sans Thai when rendering Thai characters. No conditional logic or language detection is needed — the font stack handles it natively.

### 3.2 Type Scale

ECHO uses a tight type scale optimized for data-dense interfaces.

| Token | Size | Weight | Line Height | Usage |
|---|---|---|---|---|
| `display` | 30px (text-3xl) | 700 (bold) | 1.2 | Page titles, hero headlines |
| `heading-lg` | 22px (text-xl + custom) | 600 (semibold) | 1.3 | Section headings, modal titles |
| `heading` | 18px (text-lg) | 600 (semibold) | 1.4 | Card headings, sidebar section headers |
| `heading-sm` | 15px (text-sm + custom) | 600 (semibold) | 1.4 | Sub-section headings, table group headers |
| `body` | 14px (text-sm) | 400 (normal) | 1.5 | Default body text, descriptions |
| `body-medium` | 14px (text-sm) | 500 (medium) | 1.5 | Table cell text, form labels, navigation items |
| `caption` | 12px (text-xs) | 400 (normal) | 1.5 | Timestamps, secondary info, help text |
| `caption-medium` | 12px (text-xs) | 500 (medium) | 1.5 | Badges, tags, small labels |
| `overline` | 11px (custom) | 600 (semibold) | 1.4 | All-caps labels, section overlines |
| `mono` | 13px (custom) | 400 (normal) | 1.5 | IDs, prices, numerical data |

### 3.3 Typography Rules

- **Default body text** in the dashboard is `14px` (Tailwind `text-sm`). This is intentionally smaller than typical websites because ECHO is a dense data application.
- **Headings** never use `font-bold` (700). Use `font-semibold` (600) for headings. Bold is reserved for emphasis within body text.
- **Tabular numbers:** Enable `font-variant-numeric: tabular-nums` (Tailwind: `tabular-nums`) on all columns containing numbers (prices, sizes, counts). This ensures digits align vertically in tables.
- **Text truncation:** Long text in table cells should truncate with ellipsis (`truncate` class). Full text visible on hover via tooltip.
- **No underlines on links** in the dashboard. Links are identified by orange color and hover state. Underlines are used only on the public website for accessibility.

## 4. Spacing & Layout

### 4.1 Spacing Scale

Use Tailwind's default spacing scale consistently. ECHO's key spacing tokens:

| Token | Tailwind | px | Usage |
|---|---|---|---|
| `space-0.5` | `p-0.5` | 2px | Tight padding inside tiny badges |
| `space-1` | `p-1` | 4px | Icon padding, gap between inline elements |
| `space-1.5` | `p-1.5` | 6px | Small button padding, badge padding |
| `space-2` | `p-2` | 8px | Table cell padding, compact card padding |
| `space-3` | `p-3` | 12px | Default card padding, form field padding |
| `space-4` | `p-4` | 16px | Section padding, modal padding |
| `space-6` | `p-6` | 24px | Page section gaps, large card padding |
| `space-8` | `p-8` | 32px | Major section separation |
| `space-12` | `p-12` | 48px | Page-level top/bottom padding |

### 4.2 Layout Grid

```
Dashboard Layout:
┌──────────┬──────────────────────────────────┐
│          │          Top Header (h-14)        │
│ Sidebar  ├──────────────────────────────────┤
│ (w-60)   │                                  │
│          │        Content Area              │
│          │        (p-6, max-w-full)         │
│          │                                  │
│          │                                  │
│          │                                  │
└──────────┴──────────────────────────────────┘
```

- **Sidebar width:** `w-60` (240px) when expanded, `w-16` (64px) when collapsed (icons only).
- **Top header height:** `h-14` (56px).
- **Content area padding:** `p-6` (24px) on all sides.
- **Content max width:** No max-width constraint for table views (use full width). Detail pages and settings use `max-w-4xl` (896px) or `max-w-5xl` (1024px) centered.
- **Sidebar collapse:** On screens below `lg` (1024px), sidebar collapses to icon-only mode. Below `md` (768px), sidebar becomes an overlay triggered by hamburger menu.

### 4.3 Table Layout

Tables are the core of ECHO. They must be dense enough to show many records but readable enough to scan quickly.

```
Table Dimensions:
  Row height:         h-10 (40px) — default
  Row height compact: h-8 (32px) — optional compact mode
  Cell padding:       px-3 py-2
  Header height:      h-10 (40px)
  Header bg:          bg-stone-50 (light mode)
  Row hover:          bg-stone-50 (light mode)
  Row selected:       bg-orange-50
  Border:             border-b border-stone-200 between rows
  Group header:       bg-stone-100, font-semibold, text-sm
  Sticky header:      sticky top-0 z-10
  Sticky first col:   sticky left-0 (for checkbox column)
```

## 5. Border Radius & Shadows

### 5.1 Border Radius

ECHO uses moderately rounded corners — enough to feel modern, not so much that it feels bubbly.

| Element | Radius | Tailwind |
|---|---|---|
| Buttons | 8px | `rounded-lg` |
| Cards | 12px | `rounded-xl` |
| Modals / Dialogs | 16px | `rounded-2xl` |
| Badges / Tags | 6px | `rounded-md` |
| Avatar (square) | 8px | `rounded-lg` |
| Avatar (circle) | 9999px | `rounded-full` |
| Input fields | 8px | `rounded-lg` |
| Dropdown menus | 12px | `rounded-xl` |
| Table container | 12px | `rounded-xl` |
| Tooltip | 8px | `rounded-lg` |
| Sidebar | 0 (flush to edge) | `rounded-none` |

### 5.2 Shadows

Shadows in ECHO are subtle and layered. Inspired by Attio's approach: shadows create depth without heaviness.

| Element | Shadow | Tailwind |
|---|---|---|
| Cards (resting) | `0 1px 2px rgba(0,0,0,0.05)` | `shadow-sm` |
| Cards (hover) | `0 4px 12px rgba(0,0,0,0.08)` | `shadow-md` (custom) |
| Modals | `0 20px 60px rgba(0,0,0,0.15)` | `shadow-xl` |
| Dropdowns | `0 8px 24px rgba(0,0,0,0.12)` | `shadow-lg` |
| Sidebar | `1px 0 0 var(--border)` | Border only, no shadow |
| Sticky header | `0 1px 0 var(--border)` | Border only, no shadow |
| Floating action button | `0 4px 12px rgba(0,0,0,0.15)` | `shadow-md` |
| Toast notifications | `0 8px 24px rgba(0,0,0,0.12)` | `shadow-lg` |

### 5.3 Borders

- Default border color: `border-stone-200` (light mode), `border-stone-700` (dark mode).
- Border width: `border` (1px) for all elements. Never use 2px borders.
- Dividers: use `border-b border-stone-200` or `<Separator />` from shadcn.
- Focus ring: `ring-2 ring-orange-500 ring-offset-2` on all focusable elements.

## 6. Iconography

### 6.1 Icon Library

Use **Lucide React** (`lucide-react`) as the primary icon library. It's included in shadcn/ui and covers all needed icons.

### 6.2 Icon Sizing

| Context | Size | Tailwind |
|---|---|---|
| Inline with body text | 16px | `w-4 h-4` |
| Buttons (with text) | 16px | `w-4 h-4` |
| Buttons (icon only) | 18px | `w-[18px] h-[18px]` |
| Navigation sidebar | 20px | `w-5 h-5` |
| Empty states | 48px | `w-12 h-12` |
| Feature illustrations | 64px | `w-16 h-16` |

### 6.3 Icon Rules

- Icons are always `stroke-width={1.75}` (slightly thinner than Lucide's default of 2, for a more refined look).
- Icon color follows text color — use `text-stone-500` for secondary icons, `text-stone-800` for primary, `text-orange-500` for active/selected.
- Never use colored fill icons in the dashboard. All icons are outline/stroke style.
- Pair icons with text labels in the sidebar and on buttons. Icon-only buttons require a tooltip.

## 7. Motion & Animation

Inspired by Attio's tactile, smooth interactions. Motion should feel responsive and natural, never decorative.

### 7.1 Duration & Easing

| Type | Duration | Easing | Usage |
|---|---|---|---|
| Micro | 100ms | `ease-out` | Hover states, focus rings, toggles |
| Fast | 150ms | `ease-in-out` | Dropdown open/close, tooltip appear |
| Medium | 200ms | `ease-in-out` | Slide-out panels, tab transitions |
| Slow | 300ms | `cubic-bezier(0.16, 1, 0.3, 1)` | Modal open/close, page transitions |
| Spring | 400ms | `cubic-bezier(0.34, 1.56, 0.64, 1)` | Drag-and-drop snap, card sort |

**Tailwind classes:** `transition-all duration-150 ease-in-out` as default for interactive elements.

### 7.2 Animation Patterns

| Pattern | Implementation | When |
|---|---|---|
| **Hover lift** | `hover:-translate-y-0.5 hover:shadow-md transition-all` | Cards on the dashboard, listing cards on public website |
| **Row highlight** | `hover:bg-stone-50 transition-colors duration-100` | Table rows |
| **Button press** | `active:scale-[0.98] transition-transform duration-100` | All buttons |
| **Slide in from right** | `translate-x-full → translate-x-0` over 200ms | Detail panels, slide-out editors |
| **Fade up** | `opacity-0 translate-y-2 → opacity-100 translate-y-0` over 150ms | Dropdown menus, tooltips |
| **Skeleton pulse** | `animate-pulse bg-stone-200 rounded` | Loading states for all data |
| **Drag shadow** | `shadow-xl opacity-90 scale-105` | While dragging photos, sections, pipeline stages |
| **Toast entrance** | Slide in from top-right, slight bounce | Notification toasts |
| **Count up** | Number animates from 0 to value over 500ms | Dashboard metric cards |

### 7.3 Motion Rules

- **Never** use motion longer than 400ms. It should enhance, not delay.
- All animations must respect `prefers-reduced-motion` media query. Wrap in `motion-safe:` Tailwind variants.
- Loading skeletons on every data fetch — never show empty screens or spinners for more than 200ms.
- Page transitions: content area cross-fades when switching between modules (150ms).

## 10. Dark Mode

ECHO supports dark mode, toggled per user preference.

### 10.1 Dark Mode Color Mapping

| Element | Light Mode | Dark Mode |
|---|---|---|
| Page background | `stone-50` (#FAFAF9) | `stone-950` (#0C0A09) |
| Card / surface | `white` | `stone-900` (#1C1917) |
| Elevated surface | `stone-50` | `stone-800` (#292524) |
| Primary text | `stone-800` | `stone-100` |
| Secondary text | `stone-500` | `stone-400` |
| Borders | `stone-200` | `stone-700` |
| Table header bg | `stone-50` | `stone-800` |
| Table row hover | `stone-50` | `stone-800/50` |
| Sidebar bg | `stone-900` | `stone-950` |
| Sidebar text | `stone-400` | `stone-400` (no change) |
| Sidebar active | `orange-400` on `orange-500/10` | `orange-400` on `orange-500/15` |
| Input bg | `white` | `stone-800` |
| Input border | `stone-200` | `stone-700` |
| Primary button | `orange-500` | `orange-500` (no change) |
| Badge bg | `[color]-100` | `[color]-900/30` |
| Badge text | `[color]-700` | `[color]-400` |

### 10.2 Dark Mode Implementation

Use Tailwind's `dark:` variant with `class` strategy (not `media`), so users can explicitly toggle.

```js
// tailwind.config.ts
module.exports = {
  darkMode: 'class',
  // ...
}
```

Toggle adds/removes `dark` class on `<html>` element. Preference stored in localStorage and synced to user preferences in the database.

## 11. Public Website

The public-facing agent website has a different design tone than the dashboard — warmer, more spacious, and optimized for conversion.

### 11.1 Public Website Differences from Dashboard

| Aspect | Dashboard | Public Website |
|---|---|---|
